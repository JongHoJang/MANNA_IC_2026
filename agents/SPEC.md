# 컨퍼런스 선택강의 신청 서비스

## 개요

이 서비스는 이벤터스에서 컨퍼런스 티켓을 구매한 참가자가 본인이 구매한 날짜에 맞춰 선택강의를 신청하고, 신청 내역을 확인·수정할 수 있는 모바일 중심 웹앱이다.  
운영자는 어드민 페이지에서 참가자 정보를 업데이트하고, 날짜별·타임별 강의 신청 현황과 미신청자 명단을 확인할 수 있다.

## 디자인 방향

전체 디자인은 “컨퍼런스 현장에서 바로 쓰는 모바일 티켓 앱”처럼 가볍고 명확해야 한다.  
참가자는 복잡한 설명 없이 로그인 → 타임테이블 확인 → 강의 신청 → 내 일정 확인 → 신청 완료 확인까지 1분 안에 끝낼 수 있어야 한다.

색감은 디자인 메인 색은 #EECA7E / 폰트색은 #282424, 뻔한 AI SaaS 카드 UI를 피한다.  
대신 컨퍼런스 배지, 시간표, 티켓, 체크인 보드에서 영감을 받은 디자인을 사용한다.

사용자 웹앱은 모바일 전용 앱처럼 보이게 구성한다.  
PC에서 접속해도 넓게 퍼지지 않고, 중앙에 모바일 프레임 형태로 보여준다.

어드민 페이지는 감성보다 실무성이 중요하다.  
신청 현황, 미신청자, 강의별 인원 수가 빠르게 보이는 운영 대시보드 형태로 구성한다.

차별화 디자인 아이디어:

- 사용자의 구매 날짜를 “내 패스권”처럼 보여주고, 신청 가능한 날짜만 활성화한다.
- 강의 신청 화면을 단순 목록이 아니라 “Day 1 / Time 1 / Time 2” 형태의 일정 카드로 구성한다.
- 신청 완료 후에는 모바일 티켓처럼 본인의 선택강의 내역을 보여준다.

## 기능 목록

### 기능 1: 참가자 로그인

- 설명: 참가자는 이름과 휴대폰 번호를 입력해 로그인한다.
- 사용자 스토리: 사용자는 이벤터스에서 티켓 구매 시 등록한 이름과 휴대폰 번호로 서비스에 접속할 수 있다.
- 포함 요소:
  - 이름 입력
  - 휴대폰 번호 입력
  - 참가자 DB 일치 여부 확인
  - 로그인 실패 안내
  - 로그인 성공 시 사용자 홈으로 이동
  - 어드민 계정 로그인 시 어드민 페이지로 이동

### 기능 2: 내 티켓 날짜 확인

- 설명: 로그인한 사용자가 본인이 구매한 컨퍼런스 날짜를 확인할 수 있다.
- 사용자 스토리: 사용자는 자신이 1일권, 2일권, 3일권 중 어떤 날짜를 구매했는지 확인할 수 있다.
- 포함 요소:
  - 구매한 날짜 표시
  - 신청 가능한 날짜 활성화
  - 구매하지 않은 날짜 비활성화
  - 구매하지 않은 날짜 선택 시 안내 메시지
  - 날짜별 신청 상태 표시

### 기능 3: 모바일 타임테이블

- 설명: 전체 컨퍼런스 일정을 날짜별로 확인할 수 있다.
- 사용자 스토리: 사용자는 각 날짜에 어떤 선택강의가 있고, 각 강의가 몇 번째 타임에 열리는지 볼 수 있다.
- 포함 요소:
  - Day 1 / Day 2 / Day 3 탭
  - 날짜별 강의 목록
  - 1타임 강의 목록
  - 2타임 강의 목록
  - 강의 제목
  - 강사명
  - 강의실
  - 강의 요약
  - 신청 가능 여부 표시

### 기능 4: 날짜별 선택강의 신청

- 설명: 사용자는 자신이 구매한 날짜에 한해 하루 2개의 선택강의를 신청할 수 있다.
- 사용자 스토리: 사용자는 첫 번째 타임에서 1개, 두 번째 타임에서 1개 강의를 선택해 신청할 수 있다.
- 포함 요소:
  - 구매 날짜 검증
  - 날짜별 신청 가능 여부
  - 1타임 강의 선택
  - 2타임 강의 선택
  - 동일 타임 중복 선택 방지
  - 신청 전 확인 화면
  - 신청 완료 처리
  - 이미 신청한 날짜의 상태 표시

### 기능 5: 선택강의 수정

- 설명: 사용자는 신청 마감 전까지 본인이 신청한 강의를 수정할 수 있다.
- 사용자 스토리: 사용자는 신청한 강의를 다시 확인하고, 원하는 경우 다른 강의로 변경할 수 있다.
- 포함 요소:
  - 기존 신청 내역 불러오기
  - 타임별 선택 변경
  - 수정 전 확인 모달
  - 수정 완료 메시지
  - 수정 이력 반영
  - 신청 마감 이후 수정 제한 안내

### 기능 6: 본인 신청 내역 확인

- 설명: 사용자는 자신이 신청한 모든 날짜의 선택강의 내역을 한 화면에서 확인할 수 있다.
- 사용자 스토리: 사용자는 컨퍼런스 당일 본인이 어떤 강의를 들어야 하는지 빠르게 확인할 수 있다.
- 포함 요소:
  - 날짜별 신청 내역
  - 타임별 강의 정보
  - 강의실 정보
  - 강사명
  - 신청 완료 상태
  - 미신청 날짜 안내
  - 모바일 티켓 형태의 신청 확인 화면

### 기능 7: 신청 상태 안내

- 설명: 사용자가 아직 신청하지 않은 날짜나 수정 가능한 상태를 명확히 안내한다.
- 사용자 스토리: 사용자는 자신이 무엇을 완료했고, 무엇을 아직 해야 하는지 쉽게 알 수 있다.
- 포함 요소:
  - 신청 완료
  - 미신청
  - 수정 가능
  - 수정 마감
  - 구매하지 않은 날짜
  - 상태별 배지
  - 홈 화면 요약 안내

### 기능 8: 참가자 정보 업데이트

- 설명: 어드민은 이벤터스와 연동된 스프레드시트 기반 참가자 정보를 서비스에 반영할 수 있다.
- 사용자 스토리: 운영자는 최신 티켓 구매자 정보를 업데이트해 신규 구매자도 로그인할 수 있게 만들 수 있다.
- 포함 요소:
  - 참가자 정보 업데이트 버튼
  - 업데이트 성공/실패 안내
  - 마지막 업데이트 시간 표시
  - 신규 참가자 수 표시
  - 변경된 참가자 수 표시
  - 중복 데이터 감지
  - 잘못된 휴대폰 번호 형식 표시

### 기능 9: 참가자별 신청 내역 조회

- 설명: 어드민은 참가자별로 어떤 날짜의 어떤 강의를 신청했는지 확인할 수 있다.
- 사용자 스토리: 운영자는 특정 참가자의 이름이나 휴대폰 번호로 신청 내역을 조회할 수 있다.
- 포함 요소:
  - 참가자 검색
  - 이름 검색
  - 휴대폰 번호 검색
  - 구매 날짜 확인
  - 신청한 강의 확인
  - 미신청 날짜 확인
  - 참가자 상세 보기

### 기능 10: 미신청자 명단 다운로드

- 설명: 어드민은 티켓은 구매했지만 강의를 신청하지 않은 사람들의 명단을 다운로드할 수 있다.
- 사용자 스토리: 운영자는 미신청자에게 별도 안내 문자를 보내기 위해 명단을 내려받을 수 있다.
- 포함 요소:
  - 전체 미신청자 조회
  - 날짜별 미신청자 필터
  - 이름
  - 휴대폰 번호
  - 구매 날짜
  - 미신청 날짜
  - 파일 다운로드
  - 다운로드 전 미리보기

### 기능 11: 날짜·타임별 신청 인원 통계

- 설명: 어드민은 각 날짜의 1타임, 2타임별로 강의별 신청 인원을 확인할 수 있다.
- 사용자 스토리: 운영자는 강의실 배치를 위해 어떤 강의에 몇 명이 신청했는지 확인할 수 있다.
- 포함 요소:
  - Day 1 / Day 2 / Day 3 필터
  - 1타임 / 2타임 구분
  - 강의별 신청 인원
  - 정원 대비 신청률
  - 인원 많은 순 정렬
  - 강의실 배정 참고용 보기
  - 신청 인원 다운로드

### 기능 12: 강의 정원 및 마감 관리

- 설명: 어드민은 강의별 정원을 설정하고, 정원이 찬 강의는 사용자 신청 화면에서 마감 처리할 수 있다.
- 사용자 스토리: 운영자는 강의실 수용 인원에 맞춰 신청 가능 인원을 제한할 수 있다.
- 포함 요소:
  - 강의별 정원 입력
  - 현재 신청 인원 표시
  - 잔여 좌석 표시
  - 정원 초과 신청 방지
  - 마감 상태 표시
  - 수동 마감 처리
  - 마감 강의 사용자 화면 비활성화

### 기능 13: 운영 대시보드

- 설명: 어드민 로그인 후 전체 신청 현황을 한눈에 확인할 수 있는 첫 화면을 제공한다.
- 사용자 스토리: 운영자는 현재까지의 신청률과 미신청자 규모를 빠르게 파악할 수 있다.
- 포함 요소:
  - 전체 참가자 수
  - 전체 신청 완료자 수
  - 전체 미신청자 수
  - 날짜별 신청률
  - 타임별 신청률
  - 신청 인원 상위 강의
  - 정원 초과 위험 강의
  - 최근 업데이트 시간

### 기능 14: AI 기반 운영 보조

- 설명: 운영자가 신청 현황을 더 빠르게 판단할 수 있도록 AI 요약과 추천을 제공한다.
- 사용자 스토리: 운영자는 복잡한 신청 데이터를 직접 분석하지 않아도, 어떤 강의실을 크게 배정해야 하는지 추천받을 수 있다.
- 포함 요소:
  - 날짜별 신청 현황 요약
  - 강의실 배치 추천
  - 신청률이 낮은 날짜 안내
  - 미신청자 안내 문구 초안 생성
  - 정원 초과 위험 강의 알림
  - 운영자가 확인해야 할 이슈 요약

### 기능 15: 공지 및 안내 영역

- 설명: 참가자에게 강의 신청 마감일, 수정 가능 기간, 현장 안내를 보여준다.
- 사용자 스토리: 사용자는 서비스 안에서 강의 신청과 관련된 주요 안내를 확인할 수 있다.
- 포함 요소:
  - 신청 마감일 안내
  - 수정 가능 기간 안내
  - 컨퍼런스 장소 안내
  - 현장 체크인 안내
  - 강의실 이동 안내
  - 긴급 공지 표시

## 핵심 정책

### 신청 가능 조건

- 참가자는 본인이 구매한 날짜의 강의만 신청할 수 있다.
- 구매하지 않은 날짜의 강의는 볼 수는 있지만 신청할 수 없다.
- 하루에는 1타임 1개, 2타임 1개만 신청할 수 있다.
- 날짜별로 신청 내역이 독립적으로 관리된다.

### 로그인 조건

- 참가자 로그인은 이름과 휴대폰 번호가 참가자 DB와 일치해야 한다.
- 어드민 계정은 일반 참가자와 구분된다.
- 어드민 계정으로 로그인하면 사용자 웹앱이 아니라 어드민 페이지로 이동한다.

### 데이터 기준

- 참가자 데이터는 이벤터스 구매자 정보와 연동된 스프레드시트 데이터를 기준으로 한다.
- 참가자 데이터에는 이름, 휴대폰 번호, 구매 날짜 정보가 포함된다.
- 강의 신청 데이터에는 참가자, 날짜, 타임, 강의 정보가 포함된다.

## QA 기준

- 이름과 휴대폰 번호가 일치하는 참가자만 로그인할 수 있어야 한다.
- 구매하지 않은 날짜의 강의는 신청할 수 없어야 한다.
- 하루에 1타임 1개, 2타임 1개만 신청할 수 있어야 한다.
- 신청 완료 후 본인 신청 내역에 정확히 반영되어야 한다.
- 신청 수정 시 기존 내역이 올바르게 변경되어야 한다.
- 어드민은 전체 참가자와 신청 현황을 확인할 수 있어야 한다.
- 어드민은 미신청자 명단을 다운로드할 수 있어야 한다.
- 어드민은 날짜별, 타임별, 강의별 신청 인원을 확인할 수 있어야 한다.
- 정원이 찬 강의는 추가 신청이 제한되어야 한다.
- 참가자 정보 업데이트 후 신규 구매자가 로그인할 수 있어야 한다.
