# Manna IC Admin Console

## 개요
Manna IC 운영진이 참가자 신청 현황을 한 화면에서 확인하고, 필요한 참가자 정보를 즉시 수정하며, 선택 세션 미신청자를 빠르게 추적할 수 있는 어드민 콘솔을 설계한다. 기존 사용자용 신청 경험을 바꾸기보다, 현재 존재하는 `/admin` 흐름을 확장해 운영 실무에 필요한 조회, 검수, 다운로드 기능을 MVP 중심으로 제공한다.

## 디자인 방향
현재 어드민 화면의 차분한 패널형 대시보드 톤, 둥근 카드, 밀도 있는 정보 배치를 유지한다. 화려한 마케팅형 화면이 아니라 "운영 상황판"처럼 빠르게 훑고 바로 조치할 수 있는 분위기를 우선한다.

보라색 그라데이션, 뻔한 카드 나열식 랜딩 패턴, 장식 위주 KPI 보드는 금지한다. 대신 요약 지표, 필터 바, 표, 보조 패널이 한 호흡으로 이어지는 구조를 유지하고, "미신청자 추적"과 "세션별 혼잡도 확인"이 눈에 띄게 드러나도록 정보 우선 배치를 사용한다.

차별화 포인트는 두 가지다. 첫째, 참가자 단위와 세션 단위의 상태를 같은 화면 안에서 교차 확인할 수 있게 해 운영 판단 전환 비용을 줄인다. 둘째, 단순 미신청 여부가 아니라 Day1~Day3 중 어느 날이 비어 있는지 바로 드러나는 결손 중심 뷰를 핵심 경험으로 둔다.

## 데이터 제약 및 최소 확장 방향
- 현재 `participants` 테이블에는 `name`, `phone`, `position`, `ticket_info`, `day1`, `day2`, `day3`, `is_admin`만 있고, 요구사항의 `이메일`, `소속` 필드는 존재하지 않는다.
- 따라서 1차 MVP에서는 기존 데이터만으로 화면을 구성하되, 참가자 상세/수정/다운로드 요구를 온전히 충족하기 위해 `participants.email`, `participants.organization` 두 필드의 최소 확장을 설계 범위에 포함한다.
- 이 확장은 운영 목적상 조회/수정/다운로드에서 직접 사용되므로 우선순위가 높다. 단, 다른 인증 구조나 별도 프로필 테이블 신설까지는 MVP 범위에 포함하지 않는다.
- `participants`, `registrations`, `lectures`의 관계는 유지한다. 참가자별 선택 세션 여부는 `registrations`를 기준으로 계산하고, 강의명/정원 정보는 `lectures`에서 결합한다.

## 기능 목록

### 기능 1: 어드민 전용 진입 경로 정리
- 설명: 기존 어드민 전용 페이지 경로를 `/admin`으로 유지하고, 로그인 이후 바로 운영 대시보드로 진입하게 한다.
- 사용자 스토리: 운영자는 새로운 경로를 외울 필요 없이 기존 어드민 진입점을 그대로 사용해 관리 화면으로 들어간다.
- 포함 요소:
  - `/admin` 경로 유지
  - 기존 관리자 로그인 흐름 재사용
  - 일반 사용자 접근 시 사용자 화면으로 우회
  - 로그인 완료 후 대시보드 즉시 노출

### 기능 2: 참가자 통합 목록 테이블
- 설명: 참가자 목록을 표 형태로 제공해 운영자가 이름, 소속, 직분, 연락처, 이메일, 티켓 정보를 한 번에 확인할 수 있게 한다.
- 사용자 스토리: 운영자는 전체 참가자를 표에서 빠르게 스캔하며 필요한 사람을 찾고 정보를 비교한다.
- 포함 요소:
  - 기본 컬럼: 이름, 소속, 직분, 연락처, 이메일, 티켓 정보
  - 보조 컬럼: Day1~Day3 신청 상태, 총 신청 수
  - 데이터 부재 시 빈 값 표기 규칙
  - 모바일에서는 주요 컬럼 우선 노출, 상세는 확장형으로 축약

### 기능 3: 참가자 검색 및 다중 필터
- 설명: 참가자 표 상단에 운영 실무 중심 필터를 두어 대상을 빠르게 좁힌다.
- 사용자 스토리: 운영자는 특정 이름을 검색하거나, 특정 티켓 소지자나 미신청자만 걸러서 바로 확인한다.
- 포함 요소:
  - 이름/연락처 기반 검색
  - 티켓 정보 필터
  - 일자별 미신청 필터
  - 전체 신청 완료 여부 필터
  - 관리자 계정 제외 기본값

### 기능 4: 참가자 상세 패널
- 설명: 목록에서 특정 참가자를 선택하면 우측 패널 또는 하단 확장 영역에서 신청 상세를 보여준다.
- 사용자 스토리: 운영자는 참가자 한 명을 클릭해 기본 정보와 일자별 신청 강의를 한 번에 확인한다.
- 포함 요소:
  - 기본 정보 요약
  - Day1~Day3 참석 가능 여부(`day1~day3`)
  - Day1~Day3 슬롯별 신청 결과
  - 신청한 강의명, 강연자, 장소 표시
  - 미신청 슬롯/미신청 일자 강조

### 기능 5: 참가자 정보 수정
- 설명: 현재 DB 구조를 기준으로 수정 가능한 필드를 분리해 안전하게 편집한다.
- 사용자 스토리: 운영자는 잘못 입력된 참가자 정보를 수정하고, 저장 직후 목록과 상세 화면에서 반영된 값을 확인한다.
- 포함 요소:
  - 현재 스키마 기준 수정 가능 필드: 이름, 연락처, 직분, 티켓 정보, Day1~Day3 참석 여부
  - 최소 확장 후 추가 수정 필드: 이메일, 소속
  - 읽기 전용 처리 대상: 참가자 ID, 관리자 여부, 생성 시각
  - 저장 성공/실패 피드백
  - 수정 후 테이블/상세 패널 즉시 반영

### 기능 6: 일자별 선택 세션 신청 여부 매트릭스
- 설명: 참가자별로 Day1~Day3의 2개 선택 세션 신청 여부를 표준화된 매트릭스로 보여준다.
- 사용자 스토리: 운영자는 각 참가자가 어느 날 몇 개를 신청했는지, 어떤 강의를 골랐는지, 어디가 비어 있는지 즉시 확인한다.
- 포함 요소:
  - Day1, Day2, Day3 각각 2개 슬롯 기준 표시
  - 슬롯별 신청 강의명 표시
  - 미신청 상태 명확 표기
  - 하루 전체 미신청과 부분 미신청 구분
  - 목록 테이블과 상세 패널 양쪽에서 동일 의미로 노출

### 기능 7: 미신청자 추적 뷰
- 설명: 선택 세션 미신청자를 별도 영역으로 모아 운영자가 즉시 후속 연락 대상을 파악할 수 있게 한다.
- 사용자 스토리: 운영자는 전체 참가자 목록을 뒤지지 않고도 미신청자만 모아서 확인하고 후속 조치를 준비한다.
- 포함 요소:
  - 전체 미신청자 수 요약
  - 미신청 Day 표시
  - 이름, 연락처, 이메일, 티켓 정보 빠른 확인
  - 전체 미신청과 일부 Day 미신청 구분
  - 현재 필터 상태와 연동된 대상 수 표시

### 기능 8: 미신청자 다운로드
- 설명: 미신청 참가자 목록을 운영 후속 작업용 파일로 내보낸다.
- 사용자 스토리: 운영자는 미신청자 명단을 내려받아 문자, 메일, 외부 운영 시트에서 바로 활용한다.
- 포함 요소:
  - 다운로드 대상: 이름, 연락처, 이메일, 티켓 정보, 미신청 Day
  - CSV 우선 제공, 가능하면 Excel 대응 확장 여지 명시
  - 현재 적용된 필터 기준 다운로드
  - 데이터가 없을 때 빈 파일 대신 안내 메시지 제공

### 기능 9: 세션별 신청 인원 및 정원 현황
- 설명: `lectures`와 `registrations`를 결합해 각 선택 세션의 신청자 수를 집계하고, 정원이 있으면 대비 현황까지 보여준다.
- 사용자 스토리: 운영자는 어떤 세션이 여유가 있는지, 어떤 세션이 정원에 근접했는지 바로 판단한다.
- 포함 요소:
  - 강의명, 일자, 슬롯 순서, 장소
  - 신청 인원 수
  - 정원 존재 시 `신청/정원` 형식 표시
  - 정원 미설정 강의에 대한 예외 표기
  - 날짜 필터와 연동된 세션 리스트

### 기능 10: 운영 요약 지표
- 설명: 화면 상단에 핵심 운영 수치를 요약해 현재 상태를 빠르게 파악하게 한다.
- 사용자 스토리: 운영자는 페이지 진입 직후 전체 참가자 수, 신청 완료 수, 미신청자 수, 전체 신청 건수를 보고 오늘 필요한 액션을 판단한다.
- 포함 요소:
  - 전체 참가자 수
  - 신청 완료 참가자 수
  - 미신청/부분 미신청 참가자 수
  - 전체 등록 건수
  - 필요 시 가장 혼잡한 세션 또는 미신청이 많은 Day 보조 표시

### 기능 11: 즉시성 있는 데이터 반영 흐름
- 설명: 조회, 수정, 필터, 다운로드 흐름이 서로 어긋나지 않도록 상태 반영 기준을 통일한다.
- 사용자 스토리: 운영자는 참가자 정보를 수정한 직후 새로고침 없이 최신 목록과 신청 상태를 그대로 확인한다.
- 포함 요소:
  - 수정 직후 목록 재계산
  - 참가자 상세 패널 즉시 갱신
  - 미신청자 목록/다운로드 대상 자동 재산출
  - 세션별 신청 인원 집계는 참가자 수정이 아닌 등록 데이터 변경 시만 재계산된다는 규칙 명시

## MVP 우선순위
1. `/admin` 기존 화면 확장
2. 참가자 목록 테이블 + 검색/필터
3. 참가자 상세 패널 + Day1~Day3 신청 매트릭스
4. 참가자 정보 수정
5. 미신청자 추적 + CSV 다운로드
6. 세션별 신청/정원 현황 고도화

## 화면 구성 제안
- 상단: 운영 요약 지표와 날짜 필터
- 본문 좌측 메인: 참가자 목록 테이블
- 본문 우측 보조 패널: 선택한 참가자 상세 및 수정 폼
- 하단 또는 별도 섹션: 세션별 신청 인원 현황, 미신청자 다운로드 영역
- 기존 카드 톤과 정보 구조를 유지하되, 운영자가 가장 오래 머무를 영역은 표와 상세 패널이 되도록 재배치

## 검증 기준
- 운영자가 참가자 한 명의 기본 정보와 Day1~Day3 신청 현황을 10초 안에 파악할 수 있어야 한다.
- 운영자가 참가자 정보를 수정하면 목록과 상세에서 즉시 바뀐 값을 확인할 수 있어야 한다.
- 운영자가 미신청자만 필터링하고 바로 파일로 내려받을 수 있어야 한다.
- 운영자가 세션별 신청 인원과 정원 대비 상태를 날짜별로 확인할 수 있어야 한다.
- `email`, `organization`이 없는 현재 스키마에서도 화면이 깨지지 않아야 하며, 확장 후에는 해당 값이 자연스럽게 노출되어야 한다.
